    • wsl(윈도우 서브시스템)은 윈도우 10에서만 사용가능하다.
  • 구글 크롬
    • 구글 크롬은 디버깅 툴과 최신 API를 지원해준다.
  • VSC
    • Python
    • Prettier
    • ESLint
    • Material Theme
    • Material Icon Theme
  • Git
  • WSL2 & ubuntu
    • 윈도우에서 리눅스를 사용할 수 있도록 해준다.

  • Chocolately

    • Powershell에서 동작한다.

🖍️ 윈도우 터미널 커스터마이징

  • VSC에서 Remote - WSL을 설치하면, WSL을 VSC 내에서 사용할 수 있도록 해준다.
    • 이걸 사용하면 WSL 환경에서 VSC로 작업할 수 있도록 해준다.
  • Windows Terminal의 기본 프로필을 설정하면, 기본으로 사용되는 터미널을 수정할 수 있다.
    • zsh & oh my zsh도 설치해서 쓸 수 있다. Zsh 공식 홈페이지 ohmyzsh 깃허브
  • 터미널에서 code를 입력하면, wsl 내에서 VSC를 사용할 수 있다.
  • 터미널의 설정에서 글꼴, 색상, 터미널 초기 위치나 사용할 shell등 여러가지를 커스텀 할 수 있다.
  • bashrc를 통해서 환경변수를 수정했던 것처럼 root 경로의 .zshrc를 통해서 환경변수를 설정할 수 있다.
  • Powerlevel10k는 zsh의 테마 설정 도구이다.
    • 이를 통해 설정하면 터미널의 여러 설정을 해줄 수 있다.
    • p10k configure를 통해 재설정 할 수 있다.

🐧 WSL과 리눅스 콘솔 이해하기

윈도우에서 WSL로 리눅스를 다룰 때 주의할 것

  • wsl의 리눅스 부분과 윈도우 부분은 다른 루트를 가지고 있다.
  • wsl의 mnt(mount) 폴더로 접근하면, 윈도우의 파일들에 접근할 수 있다.
  • 다만 wsl에 파일을 생성한다고 윈도우에도 생성되는 것은 아니다. 접근이 가능하기 때문에 윈도우에서 파일을 생성하고 접근하는 것이 좋다.
  • 반대로 윈도우에서 리눅스 파일을 건드리면 문제가 생길 수 있으니 건드리지 않는 것이 좋다.

PPA와 WSL의 Python

  • WSL의 많은 부분이 python2.7에 의존하기 때문에 지우면 안된다.
  • apt는 패키지를 설치할 수 있도록 도와주지만, 우리의 데이터베이스에 해당 패키지가 없을 수 있다. 이런 때에는 PPA(personal package archive)를 추가한다.
  • deadsnake는 이러한 ppa의 하나이다.

deadsnakes로 python 설치하기

  • apt-get update는 ppa가 추가된 등의 상황에서 apt의 데이터베이스를 갱신한다.
  • zshrc에 alias를 추가하면 명령어의 별칭을 만들어줄 수 있다.

WSL과 윈도우의 환경의 차이

  • WSL의 VSC에는 확장프로그램을 별도로 설정해야한다.
  • 종종 WSL의 리눅스에만 설치하면 정상동작하지 않고, 윈도우에 설치하고 접근해야 정상동작하는 경우가 있다. (MongoDB가 그렇다)
